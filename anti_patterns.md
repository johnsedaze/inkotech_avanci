# Анти-паттерны и рекомендации по производительности 1С

Консолидированный файл анти-паттернов, проблем производительности и нарушений best practices для разработки 1С. Ссылайся на этот файл вместо дублирования примеров.

## Критические анти-паттерны (обязательно исправлять)

### 1. Запрос в цикле

**Влияние:** O(n) обращений к БД вместо O(1)
**Серьёзность:** CRITICAL

```bsl
// CRITICAL: N обращений к БД
Для Каждого Строка Из Данные Цикл
    Запрос = Новый Запрос("ВЫБРАТЬ ... ГДЕ Ссылка = &Ссылка");
    Запрос.УстановитьПараметр("Ссылка", Строка.Ссылка);
    РезультатЗапроса = Запрос.Выполнить();
КонецЦикла;

// Оптимально: 1 обращение к БД
Запрос = Новый Запрос;
Запрос.Текст =
"ВЫБРАТЬ ...
|ГДЕ
|   Ссылка В (&СписокСсылок)";
Запрос.УстановитьПараметр("СписокСсылок",
    Данные.ВыгрузитьКолонку("Ссылка"));
РезультатЗапроса = Запрос.Выполнить();
```

### 2. Обращение к реквизитам через точку

**Влияние:** загрузка всего объекта из БД
**Серьёзность:** CRITICAL

```bsl
// CRITICAL: полная загрузка объекта для каждого реквизита
ИНН = Контрагент.ИНН;
КПП = Контрагент.КПП;
Наименование = Контрагент.Наименование;

// Оптимально: один целевой запрос через БСП
Реквизиты = ОбщегоНазначения.ЗначенияРеквизитовОбъекта(
    Контрагент, "ИНН, КПП, Наименование");
ИНН = Реквизиты.ИНН;
КПП = Реквизиты.КПП;
Наименование = Реквизиты.Наименование;
```

**Методы БСП:**

| Метод | Назначение |
|-------|-----------|
| `ОбщегоНазначения.ЗначениеРеквизитаОбъекта` | Один реквизит одной ссылки |
| `ОбщегоНазначения.ЗначенияРеквизитовОбъекта` | Несколько реквизитов одной ссылки |
| `ОбщегоНазначения.ЗначениеРеквизитаОбъектов` | Один реквизит нескольких ссылок |

### 3. Подзапрос в SELECT

**Влияние:** выполнение N+1 запросов
**Серьёзность:** CRITICAL

```bsl
// CRITICAL: подзапрос выполняется для каждой строки
"ВЫБРАТЬ
|   Заказы.Ссылка,
|   (ВЫБРАТЬ СУММА(Оплаты.Сумма)
|    ИЗ Документ.Оплата КАК Оплаты
|    ГДЕ Оплаты.Заказ = Заказы.Ссылка) КАК СуммаОплат
|ИЗ
|   Документ.Заказ КАК Заказы"

// Оптимально: временная таблица + соединение (см. query-optimization-tips.md)
"ВЫБРАТЬ
|   Оплаты.Заказ КАК Заказ,
|   СУММА(Оплаты.Сумма) КАК СуммаОплат
|ПОМЕСТИТЬ ВТИтогиОплат
|ИЗ
|   Документ.Оплата КАК Оплаты
|СГРУППИРОВАТЬ ПО
|   Оплаты.Заказ
|ИНДЕКСИРОВАТЬ ПО
|   Заказ
|;
|ВЫБРАТЬ
|   Заказы.Ссылка КАК Ссылка,
|   ЕСТЬNULL(ИтогиОплат.СуммаОплат, 0) КАК СуммаОплат
|ИЗ
|   Документ.Заказ КАК Заказы
|       ЛЕВОЕ СОЕДИНЕНИЕ ВТИтогиОплат КАК ИтогиОплат
|       ПО Заказы.Ссылка = ИтогиОплат.Заказ"
```

## Высокий приоритет

### 4. Фильтр виртуальной таблицы в ГДЕ

**Влияние:** полный перебор таблицы вместо использования индекса
**Серьёзность:** HIGH

```bsl
// HIGH: фильтр после вычисления виртуальной таблицы
"ВЫБРАТЬ
|   Остатки.Номенклатура КАК Номенклатура,
|   Остатки.КоличествоОстаток КАК Остаток
|ИЗ
|   РегистрНакопления.ТоварыНаСкладах.Остатки() КАК Остатки
|ГДЕ
|   Остатки.Склад = &Склад"

// Оптимально: фильтр в параметрах виртуальной таблицы
"ВЫБРАТЬ
|   Остатки.Номенклатура КАК Номенклатура,
|   Остатки.КоличествоОстаток КАК Остаток
|ИЗ
|   РегистрНакопления.ТоварыНаСкладах.Остатки(, Склад = &Склад) КАК Остатки"
```

### 5. Отсутствие ПЕРВЫЕ N

**Влияние:** загрузка всех записей, когда нужна только часть
**Серьёзность:** HIGH

```bsl
// HIGH: загрузка всех записей
"ВЫБРАТЬ
|   Контрагенты.Ссылка КАК Ссылка
|ИЗ
|   Справочник.Контрагенты КАК Контрагенты"

// Оптимально: ограничение на уровне запроса
"ВЫБРАТЬ ПЕРВЫЕ 10
|   Контрагенты.Ссылка КАК Ссылка
|ИЗ
|   Справочник.Контрагенты КАК Контрагенты"
```

### 6. Избыточные клиент-серверные вызовы

**Влияние:** сетевые задержки, сериализация контекста
**Серьёзность:** HIGH

```bsl
// HIGH: множественные серверные вызовы
&НаКлиенте
Процедура Обработать(Команда)
    Данные1 = ПолучитьДанные1НаСервере();
    Данные2 = ПолучитьДанные2НаСервере();
    Данные3 = ПолучитьДанные3НаСервере();
КонецПроцедуры

// Оптимально: один серверный вызов
&НаКлиенте
Процедура Обработать(Команда)
    ВсеДанные = ПолучитьВсеДанныеНаСервере();
КонецПроцедуры

&НаСервереБезКонтекста
Функция ПолучитьВсеДанныеНаСервере()
    Результат = Новый Структура;
    Результат.Вставить("Данные1", ПолучитьДанные1());
    Результат.Вставить("Данные2", ПолучитьДанные2());
    Результат.Вставить("Данные3", ПолучитьДанные3());
    Возврат Результат;
КонецФункции
```

### 7. Использование &НаСервере вместо &НаСервереБезКонтекста

**Влияние:** ненужная передача контекста формы
**Серьёзность:** HIGH

```bsl
// HIGH: передача полного контекста формы
&НаСервере
Функция ПолучитьДанныеНаСервере()
    Возврат ВыполнитьЗапрос();
КонецФункции

// Оптимально: без передачи контекста
&НаСервереБезКонтекста
Функция ПолучитьДанныеНаСервере(Параметры)
    Возврат ВыполнитьЗапрос(Параметры);
КонецФункции
```

## Средний приоритет

### 8. Отсутствие кеширования

**Влияние:** повторение дорогих операций
**Серьёзность:** MEDIUM

```bsl
// MEDIUM: одинаковые вычисления повторяются
Для Каждого Строка Из ТаблицаДанных Цикл
    Курс = ПолучитьКурсВалюты(Строка.Валюта, Строка.Дата);
КонецЦикла;

// Оптимально: кеширование результатов
КэшКурсов = Новый Соответствие;

Для Каждого Строка Из ТаблицаДанных Цикл

    Ключ = Строка.Валюта + "|" + Формат(Строка.Дата, "ДФ=yyyyMMdd");
    Курс = КэшКурсов.Получить(Ключ);

    Если Курс = Неопределено Тогда
        Курс = ПолучитьКурсВалюты(Строка.Валюта, Строка.Дата);
        КэшКурсов.Вставить(Ключ, Курс);
    КонецЕсли;

КонецЦикла;
```

### 9. Алгоритм O(n²)

**Влияние:** экспоненциальная деградация производительности
**Серьёзность:** MEDIUM

```bsl
// MEDIUM: O(n²) вложенный цикл поиска
Для Каждого Строка1 Из Таблица1 Цикл
    Для Каждого Строка2 Из Таблица2 Цикл
        Если Строка1.Ключ = Строка2.Ключ Тогда
            // Обработка совпадения
        КонецЕсли;
    КонецЦикла;
КонецЦикла;

// Оптимально: O(n) через Соответствие
ИндексТаблицы2 = Новый Соответствие;
Для Каждого Строка2 Из Таблица2 Цикл
    ИндексТаблицы2.Вставить(Строка2.Ключ, Строка2);
КонецЦикла;

Для Каждого Строка1 Из Таблица1 Цикл
    Строка2 = ИндексТаблицы2.Получить(Строка1.Ключ);
    Если Строка2 <> Неопределено Тогда
        // Обработка совпадения
    КонецЕсли;
КонецЦикла;
```

### 10. Глубокая вложенность

**Влияние:** плохая читаемость, сложность сопровождения
**Серьёзность:** MEDIUM

**Принцип:** предпочитать ранний выход (guard clause) для снижения вложенности. Проверяй негативные условия в начале и выходи из процедуры/функции через `Возврат` или из итерации цикла через `Продолжить`.

```bsl
// MEDIUM: глубокая вложенность (>4 уровней)
Если Условие1 Тогда
    Если Условие2 Тогда
        Если Условие3 Тогда
            Если Условие4 Тогда
                // Логика
            КонецЕсли;
        КонецЕсли;
    КонецЕсли;
КонецЕсли;

// Оптимально: ранний выход
Если НЕ Условие1 Тогда
    Возврат;
КонецЕсли;

Если НЕ Условие2 Тогда
    Возврат;
КонецЕсли;

Если НЕ Условие3 Тогда
    Возврат;
КонецЕсли;

Если НЕ Условие4 Тогда
    Возврат;
КонецЕсли;

// Логика
```

Ранний выход в цикле через `Продолжить`:

```bsl
// Плохо: тело цикла обёрнуто в условие
Для Каждого Строка Из ТаблицаДанных Цикл
    Если Строка.Активна Тогда
        Если ЗначениеЗаполнено(Строка.Контрагент) Тогда
            // Основная логика обработки строки
            ОбработатьСтроку(Строка);
        КонецЕсли;
    КонецЕсли;
КонецЦикла;

// Хорошо: ранний выход через Продолжить
Для Каждого Строка Из ТаблицаДанных Цикл
    Если НЕ Строка.Активна Тогда
        Продолжить;
    КонецЕсли;
    Если НЕ ЗначениеЗаполнено(Строка.Контрагент) Тогда
        Продолжить;
    КонецЕсли;
    // Основная логика обработки строки
    ОбработатьСтроку(Строка);
КонецЦикла;
```

## Архитектурные анти-паттерны

### Большой клубок грязи (Big Ball of Mud)
- Нет чёткой структуры
- Всё зависит от всего
- **Влияние**: невозможность сопровождения, высокий риск изменений

### Бог-модуль (God Module)
- Один модуль делает всё
- Сотни процедур в одном модуле
- **Влияние**: сложно понять, тестировать, модифицировать

### Жёсткая связанность (Tight Coupling)
- Модули напрямую зависят от деталей реализации
- Изменения каскадируются между модулями
- **Влияние**: высокая стоимость модификации

### Копипаст-архитектура
- Один и тот же код в нескольких местах
- Нет общих модулей
- **Влияние**: несогласованность, бремя сопровождения

### Преждевременная оптимизация
- Сложное кеширование до доказанной необходимости
- Переусложнение для текущего масштаба
- **Влияние**: ненужная сложность

## Оптимизированные паттерны

### Пакетный запрос с временной таблицей

```bsl
МенеджерВТ = Новый МенеджерВременныхТаблиц;

Запрос = Новый Запрос;
Запрос.МенеджерВременныхТаблиц = МенеджерВТ;

// Шаг 1: создать временную таблицу с входными данными
Запрос.Текст =
"ВЫБРАТЬ
|   Данные.Номенклатура КАК Номенклатура,
|   Данные.Склад КАК Склад
|ПОМЕСТИТЬ ВТ_Входные
|ИЗ
|   &ТаблицаДанных КАК Данные";
Запрос.УстановитьПараметр("ТаблицаДанных", ТаблицаДанных);
Запрос.Выполнить();

// Шаг 2: соединение с регистром для пакетного результата
Запрос.Текст =
"ВЫБРАТЬ
|   ВТ_Входные.Номенклатура КАК Номенклатура,
|   ВТ_Входные.Склад КАК Склад,
|   ЕСТЬNULL(Остатки.КоличествоОстаток, 0) КАК Остаток
|ИЗ
|   ВТ_Входные КАК ВТ_Входные
|       ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.ТоварыНаСкладах.Остатки(
|           ,
|           (Номенклатура, Склад) В
|               (ВЫБРАТЬ ВТ.Номенклатура, ВТ.Склад ИЗ ВТ_Входные КАК ВТ)
|       ) КАК Остатки
|       ПО ВТ_Входные.Номенклатура = Остатки.Номенклатура
|           И ВТ_Входные.Склад = Остатки.Склад";
РезультатЗапроса = Запрос.Выполнить();
```

### Пакетное получение реквизитов через БСП

```bsl
// Вместо отдельных вызовов в цикле
СписокКонтрагентов = ТаблицаДанных.ВыгрузитьКолонку("Контрагент");

// Получить все реквизиты одним вызовом
ТаблицаРеквизитов = ОбщегоНазначения.ЗначенияРеквизитовОбъектов(
    СписокКонтрагентов, "ИНН, КПП, Наименование");

// Построить индекс для поиска
СоответствиеРеквизитов = Новый Соответствие;
Для Каждого СтрокаРеквизитов Из ТаблицаРеквизитов Цикл
    СоответствиеРеквизитов.Вставить(
        СтрокаРеквизитов.Ссылка, СтрокаРеквизитов);
КонецЦикла;
```

## Краткая таблица проверки

| Анти-паттерн | Серьёзность | Что искать |
|-------------|-------------|------------|
| Запрос в цикле | CRITICAL | `Для Каждого` с `Новый Запрос` внутри |
| Обращение через точку | CRITICAL | `.Реквизит` у ссылочных типов |
| Подзапрос в SELECT | CRITICAL | Вложенный `ВЫБРАТЬ` в списке полей |
| Фильтр ВТ в ГДЕ | HIGH | Условия на результатах виртуальных таблиц |
| Нет ПЕРВЫЕ N | HIGH | Большие запросы без `ПЕРВЫЕ` |
| Множественные серверные вызовы | HIGH | Последовательные `НаСервере`-вызовы с клиента |
| &НаСервере вместо БезКонтекста | HIGH | Серверный вызов, не использующий контекст формы |
| Нет кеширования | MEDIUM | Повторные дорогие вызовы с одинаковыми параметрами |
| Циклы O(n²) | MEDIUM | Вложенные циклы поиска совпадений |
| Глубокая вложенность | MEDIUM | >4 уровней условий/циклов |
