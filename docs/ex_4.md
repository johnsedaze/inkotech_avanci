## Тест-кейс 4: Уменьшение суммы заказа с фиксациями — Этап3 не должен затрагиваться

### Контекст
Три этапа. После первого закрытия периода (фиксации по Этап1 и Этап2)
сумма заказа уменьшается. При пересчёте НайденЦелевойЭтап для Оплаты 2
определяется как Этап3 (по НакопленоПосле), но правильное поведение —
обрезать целевой до ПервыйНезакрытого (Этап2), а остаток отправить в переизбыток.

Проверяет: обрезка целевого при наличии фиксации у самой оплаты
**в ПервомНезакрытом** (не дальше него) — обрезка должна сработать.

### Анализ шага 1а после уменьшения суммы

**Оплата 000027** (НакопленоДо = 0):
- ПервыйНезакрытый = Этап1 (порог 200 > 0)
- НакопленоДо = ПорогПредыдущего = 0 → **НОВАЯ ОПЛАТА**
- Целевой = Этап1

**Оплата 000028** (НакопленоДо = 300, НакопленоПосле = 600):
- ПервыйНезакрытый = Этап2 (первый порог > 300 → 400), ПервыйНезакрытыйИнд = 1
- НакопленоДо (300) > ПорогПредыдущего (200) → **ДОБИВКА**
- НакопленоПосле (600) ≤ 800 → НайденЦелевойЭтап = Этап3 (первый порог ≥ 600 = 800)
- НайденЦелевойЭтап (Этап3, №3) > ПервыйНезакрытый (Этап2, №2) → проверка обрезки

**Идея 1 — проверка МаксИндФиксОплаты для Оплаты 000028:**
- Фиксации Оплаты 000028: есть в Этап2 (индекс = 1) → МаксИндФикс = 1
- Условие обрезки: МаксИндФикс (1) ≤ ПервыйНезакрытыйИнд (1) → **ИСТИНА → обрезаем до Этап2** ✓
