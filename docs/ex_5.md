## Тест-кейс 7: Уменьшение суммы до 900 — строки ожидания для RTU-B

### Контекст
Вариант тест-кейса 6 с менее агрессивным уменьшением суммы (1000 → 900).
После уменьшения в каждом этапе образуется зазор 25 руб., который
займёт RTU-Б (100 руб., ещё не реализован). До прихода RTU-Б этот зазор
отображается как строки ожидания. Переизбыток остаётся, Этап3 не затрагивается.

---

### Входные данные

**Сумма заказа до уменьшения: 1 000**

- Этап1 (25%): лимит = 250
- Этап2 (25%): лимит = 250
- Этап3 (50%): лимит = 500

**Оплаты:**
- Оплата 000027: 300 (27.03.2026)
- Оплата 000028: 300 (29.03.2026)

**РТУ:**
- РТУ-А: 800 (28.03.2026)
  - РТУ-А × Этап1 (25%): 200
  - РТУ-А × Этап2 (25%): 200
  - РТУ-А × Этап3 (50%): 400

**Фиксации после закрытия периода (31.03.2026):**
- Оплата 000027 × РТУ-А × Этап1 = 200 | Аванс | ✓
- Оплата 000028 × РТУ-А × Этап2 = 200 | Зачёт оплаты | ✓

---

**Сумма заказа после уменьшения: 900**

- Этап1 (25%): лимит = 225
- Этап2 (25%): лимит = 225
- Этап3 (50%): лимит = 450
- Пороги: Этап1 = 225 / Этап2 = 450 / Этап3 = 900

РТУ-А лимиты по этапам (не изменились — зависят от суммы РТУ):
- РТУ-А × Этап1 (25%): 200
- РТУ-А × Этап2 (25%): 200
- РТУ-А × Этап3 (50%): 400

**Ожидаемый РТУ-Б (ещё не реализован): 100**
- РТУ-Б × Этап1 (25%): 25  ← займёт строку ожидания Этап1
- РТУ-Б × Этап2 (25%): 25  ← займёт строку ожидания Этап2
- РТУ-Б × Этап3 (50%): 50  ← не затрагивается

---

### Анализ шага 1а после уменьшения суммы

**Оплата 000027** (НакопленоДо = 0):
- НОВАЯ ОПЛАТА → целевой = Этап1

**Оплата 000028** (НакопленоДо = 300, НакопленоПосле = 600):
- ПервыйНезакрытый = Этап2 (порог 450 > 300), ПервыйНезакрытыйИнд = 1
- ДОБИВКА → НайденЦелевойЭтап = Этап3 (первый порог ≥ 600 = 900)

**Идея 1 — проверка для Оплаты 000028:**
- Фиксации Оплаты 000028: Этап2 (индекс = 1) → МаксИндФикс = 1
- МаксИндФикс (1) ≤ ПервыйНезакрытыйИнд (1) → **обрезка до Этап2** ✓

---

### Пошаговый расчёт Фазы 1

**Оплата 000027** (целевой = Этап1):
- ЗафиксированоПоОплате = 200 → ОстатокПлатежа = 300 − 200 = **100**
- Этап1: лимит=225, фиксации=200, авто=0 → ёмкость=**25**
  - СуммаНаЭтап = min(100, 25) = 25
  - РТУ-А в Этап1: лимит=200, распределено=200 → остаток=0 → **База = 0**
  - → строка ожидания = **25** (ждём РТУ-Б)
  - АвтоПоЭтапам[Этап1] = 25
- Остаток = 100 − 25 = **75** → переизбыток

**Оплата 000028** (целевой = Этап2):
- ЗафиксированоПоОплате = 200 → ОстатокПлатежа = 300 − 200 = **100**
- Этап1: лимит=225, фиксации=200, авто=25 → ёмкость=**0** → пропуск
- Этап2: лимит=225, фиксации=200, авто=0 → ёмкость=**25**
  - СуммаНаЭтап = min(100, 25) = 25
  - РТУ-А в Этап2: лимит=200, распределено=200 → остаток=0 → **База = 0**
  - → строка ожидания = **25** (ждём РТУ-Б)
- Остаток = 100 − 25 = **75** → переизбыток

- ✓ Фиксации не изменены (2 строки с галочкой)
